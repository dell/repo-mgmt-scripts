Pre-Setup

Important! You must complete a couple of pre-setup tasks before using this repository, as outlined below.
  • Import vendor GPG keys: Red Hat Enterprise Linux 3/4 only 
    rpm --import /usr/share/rhn/RPM-GPG-KEY
  • Update System: We recommend that you update your system before installing.
    • Red Hat Enterprise Linux systems: 
      up2date -u
    • Fedora Core systems: 
      yum update
    • SUSE systems:...(need info here).

Note: If you are a Dell internal tester, you cannot reach the Red Hat Network through the firewall. Go to http://rhn.linuxdev.us.dell.com instead. You can register your machine there and update your system before running through the following steps.

Set Up

To set up your system to pull from the Dell yum repository, run the following command as root: 
wget -q -O - http:///bootstrap.cgi | bash

For information specific to setup of libsmbios, please see the libsmbios page

OpenManage Server Administrator

The OMSA RPMS have moved into the hardware repository. Please see the instructions in the hardware repository for installation instructions for OMSA.

Support

This is the unofficial Dell yum package repository. Note that this repository is not officially Dell-supported. For support questions related to this package repository, please use the appropriate mailing list:
  • For libsmbios related questions: libsmbios-devel Note that this is the official repository for libsmbios releases. All other software in this repository is unofficial. For libsmbios questions, please use the mailing list.
  • For other questions (OMSA, etc): linux-poweredge
Do not call Dell Technical support with questions about this repository, or software installed from this repository, as they will not be able to help you! For all questions, please use the mailing lists above.

Local Mirrors of this site

If you wish to mirror this site to a server inside your firewall, please be aware that rsync is not available on this server. We recommend using the 'reposync' application which is included in the yum-utils package of the Fedora Project. You may also be interested in fedora-mirror for maintaining your own mirror of the Fedora packages when rsync isn't possible for you.
